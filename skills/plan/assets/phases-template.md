## Key Learnings Protocol

### For dev Agents
1. Before starting Phase N, read ALL key-learnings files from Phase 0 through Phase N-1
2. After completing Phase N, create `key-learnings/key-learnings-{NN}.md`
3. If your implementation diverges from this file, document the divergence in key-learnings
4. Key-learnings take precedence over this file when they conflict — they reflect actual implementation reality

### For qa Agents
1. Read the phase's key-learnings file before validating
2. After validation, update the key-learnings file with any issues found and resolutions applied
3. Add a "qa Notes" section to the key-learnings file

### Key Learnings File Location
`./key-learnings/key-learnings-{NN}.md`

Where `{NN}` is the zero-padded phase number (00, 01, 02, ...).

### Conflict Resolution
If a key-learnings file contradicts this document:
- Key-learnings wins (it reflects what was actually built)
- The next phase must follow the key-learnings version
- Document the divergence clearly for traceability
